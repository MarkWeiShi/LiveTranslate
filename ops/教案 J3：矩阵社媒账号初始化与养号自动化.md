# 教案 J3：矩阵社媒账号初始化与养号自动化（魔法哈奇 · 童年怀旧）

> 课程目标：把"批量创建一批社媒账号 + 养号"这件事，从一次性人工操作，沉淀为一套**可批量复制、AI 驱动、表格可追溯**的"账号矩阵初始化与内容养号"工作流。
> 内容母题：**魔法哈奇 / 哈奇小镇** 的童年元素与回忆；目标人群：**90 年代末—00 后**的怀旧用户。
> 主战平台：**百度贴吧 · 抖音 · 小红书 · 知乎 · B站**。
> 文档版本：v2.0

---

## 0. 合规前置（先读，决定成败）

本教案里的"**养号**" = 用**真实、优质、垂直**的内容，把账号的**算法权重与社区可信度**养起来；靠的是稳定更新 + 真实互动 + 内容价值，**不是**刷量 / 互赞群 / 外挂 / 批量黑产号 / 虚假流量。

- 五大平台都**支持品牌与创作者做矩阵**（企业号 / 蓝 V / 官方认证 / MCN 矩阵）——在规则内做矩阵是被允许的。
- 账号一律用**自有实名**注册；内容**原创或获授权**；魔法哈奇相关素材需**品牌授权**。
- **不做**：批量买号、群控刷量、机器互动、机械搬运。这些在国内平台是高危红线，封号连坐、且本就跑不出长期复利。
- 一句话：**矩阵 = 多个垂直人设号 × 真实怀旧内容**，不是"多注册几个号去刷"。本教案所有"自动化"都指**内容生产 / 排期分发 / 数据复盘**的自动化，不指伪造真人行为。

## 1. 定位（Positioning）

本教案解决矩阵运营中最底层、最重的"基建"：**从 0 到 N 批量起号 + 合规养号**，用魔法哈奇怀旧内容唤醒老玩家、聚拢同代怀旧人群，沉淀社群并为 3D 哈奇 / 小程序回流蓄水。

三句话定位：

- **这是投资行为，不是消费行为。** 每个矩阵号都是可沉淀、可二次触达的**内容资产 + 私域入口**，账号 / 粉丝 / 内容版权最终都要进公司统一管理，而不是发完即弃。
- **人定标准 + 对内容质量与合规负责，其余尽量 AI 化。** 账号人设、昵称、头像、简介、文案、图、视频脚本这些"内容生产"交给 AI；人设标准、选题方向、合规与版权判断由人兜底。
- **一套 prompt + 一套 SOP 跑出一切。** 关键不在某一个号，而在"**账号人设标准 + 怀旧内容母题库 + 各平台改造规范**"。复制到新号 / 新平台，只换参数不换流程。

适用边界：本教案聚焦 **魔法哈奇怀旧矩阵**，方法论同样适用于任意怀旧 IP / 其他人群（见第 9 节）。

## 4. 详细工作流（七步）

> 总链路：**账号规划 → AI 生成账号身份 → 合规注册 → 资料初始化 → 养号 SOP → 台账管理 → 内容生产与分发**

### 步骤 1：账号规划（Account Planning）

- **操作**：为每个平台确定人设母题、内容支柱（3–4 个）、目标受众、与同矩阵其他号的差异化。例：抖音"老场景复刻号"vs"童年游戏盘点号"做区隔，避免自我同质化。
- **AI**：大模型做人设与选题 brainstorm。
- **prompt 示例**：
    > "你是怀旧向社媒矩阵策划。围绕【魔法哈奇/哈奇小镇】童年怀旧，面向 90 末—00 后，要在抖音做 3 个差异化账号。请为每个账号输出：① 人设一句话 ② 受众画像 ③ 3 个内容支柱 ④ 与其他两个号的差异点。markdown 表格。"
- **产出**：《矩阵账号规划表》（平台 × 人设 × 受众 × 内容支柱）。

### 步骤 2：AI 生成账号身份（Identity Generation）

- **操作**：为每个号批量生成「昵称 + 主页简介 + 头像描述」。昵称要怀旧语感、与人设一致、不撞名；头像统一怀旧风（可含哈奇元素，需授权）；简介带母题关键词 + 轻互动钩子（如"评论区报到你哪年玩的"）。
- **AI**：大模型出昵称/简介候选 → 生图工具按描述出头像。
- **产出**：每号一份 `identity.json`（见第 6 节模板）。

### 步骤 3：合规注册（Registration）

- **操作**：用**自有实名手机号**在各平台**官方注册**；优先开通**企业号 / 专业号 / 创作者认证**（享受矩阵管理、数据、合规分发能力）。一号一档、资料填全。
- **红线**：不买成品号 / 黑产号、不一套设备批量注册作弊（会连坐封号，且违反平台协议）。
- **产出**：可登录的合规账号，录入台账。

### 步骤 4：资料初始化（Profile Setup）

- **操作**：头像 + 昵称 + 简介 + 主页装修 + 合集/置顶 + 背景图，**垂直定位一眼可辨**（"这是个讲童年游戏怀旧的号"）。先把"门面"做完整再发内容，利于算法判定垂类与初始推荐。
- **产出**：完成度高的主页（资料完善是养号第一步）。

### 步骤 5：养号 SOP（合规冷启动 → 稳定运营）

> 原则：**靠真实优质内容养算法权重**，节奏自然、垂直深耕，不刷量、不机械高频。

- **第 1–3 天（定调）**：完善资料；垂直浏览同领域优质内容，真实点赞/收藏/关注少量同好；不急着发。
- **第 4–7 天（轻互动 + 试水）**：每天真实逛 10–15 分钟，留 1–2 条真诚评论；发 1 条轻量内容（如一张怀旧截图 + 一句"还记得吗"）试水反馈。
- **第 8–14 天（首发主内容）**：发布第 1–3 条主内容（场景复刻/盘点/混剪，AI 辅助生产），每 2–3 天 1 条，观察完播/互动调方向。
- **第 15 天+（稳定运营）**：按平台节奏稳定更新（短视频号可日更/隔日更，图文/长文 2–3 条/周），固定优质时段发布，**真实回复评论、经营评论区社群感**。
- **全程**：垂直不跑偏；用数据迭代选题；**不参与互赞/刷量/搬运**——这些会被限流且毁号。

- **AI**：大模型生成脚本/文案/评论区话术（真人风格、由人审），生图/剪辑产出内容。
- **prompt 示例**：
    > "给【魔法哈奇童年怀旧】抖音号写 1 条 30 秒短视频脚本：开头 3 秒钩子('还有人记得哈奇小镇吗')，中间老场景复刻 + 1 个经典玩法回忆，结尾引导'评论区报到你哪年玩的'。给出分镜 + 旁白 + 卡点 BGM 建议 + 5 个话题标签。"
- **产出**：每号《养号执行记录》（日期/动作/内容链接/数据）。

### 步骤 6：台账管理（Ledger）

- **操作**：所有号的平台、人设、绑定手机号、创作者类型、内容支柱、养号阶段、更新节奏、粉丝/互动、状态统一进多维表格；敏感信息（手机号/密码）走权限控制。
- **产出**：见第 7 节字段。

### 步骤 7：内容生产与分发（Production & Distribution）

- **操作**：建"**一个母题 → 多平台改造**"的内容流水线：同一怀旧选题，按各平台形态**二次创作**（贴吧长帖 / 抖音竖屏 / 小红书九宫格 / 知乎长答 / B站混剪），**不是一稿机械搬运**（搬运会被判低质限流）。用排期工具统一发布、统一回收数据。
- **AI**：上游 AI 批量产稿 + 模板化改造；排期工具做多号分发。
- **产出**：一套选题在五平台落地为多条适配内容，产能放大、风格统一。

## 5. AI 工具与基建：配置 / 使用

### 5.1 文案 / 脚本（大模型）
- 用途：怀旧选题、标题、短视频脚本、长文提纲、笔记正文、评论区话术（真人风、人工审）。
- 用法：把第 6 节内容 prompt 参数化（平台 + 母题 + 人设），批量产稿落盘，人工精修。

### 5.2 生图 / 头像 / 九宫格
- 用途：怀旧风头像（可含**已授权**的哈奇元素）、小红书九宫格、视频封面。
- 要点：统一视觉调性（暖色/像素/老 UI 怀旧感），每号有差异。

### 5.3 生视频 / 剪辑 / 配音
- 用途：老场景复刻、童年混剪、卡点 BGM、字幕与配音。
- 要点：BGM 与素材**走授权**；魔法哈奇画面用**品牌授权素材**。

### 5.4 矩阵管理 / 排期 / 数据
- 矩阵：各平台**官方企业号/矩阵后台**统一管理多号；排期工具统一发布。
- 数据：平台自带创作者数据 + 第三方榜单工具做爆款拆解与选题迭代。
- **不使用**：群控/养号外挂/刷量工具（违规且毁号）。

### 5.5 合规与版权
- 魔法哈奇素材 → 品牌授权；BGM/影视素材 → 版权授权；老素材 → 注明出处。
- 内容遵守各平台社区规范；怀旧 ≠ 盗版/侵权；不虚假、不诱导。

## 6. Prompt 模板（可直接复制）

### 6.1 账号身份生成（批量 JSON）

```text
你是"魔法哈奇童年怀旧"社媒矩阵账号初始化助手。请为下列账号批量生成身份信息。

【输入参数】
- 平台(platform)：抖音        # 可替换为 百度贴吧/抖音/小红书/知乎/B站
- 人设方向(persona)：童年游戏考古号（魔法哈奇为主，旁及同期网游）
- 内容母题：哈奇小镇场景/经典玩法/BGM/老版本考古/同代游戏盘点
- 目标受众：90末—00后 怀旧人群、魔法哈奇老玩家
- 数量(count)：3

【硬性要求】
1. 昵称(nickname)：中文，怀旧/亲切语感，符合该平台风格，避免撞名，给 3 个备选。
2. 简介(bio)：1-2 句，含怀旧关键词 + 1 个互动钩子（如"评论区报到你哪年入坑"）+ 至多 1 个 emoji。
3. 头像描述(avatar_prompt)：中文，暖色怀旧风 + 哈奇元素（需授权），干净构图，便于 AI 生图。
4. 主页规划(profile_plan)：3 个内容支柱(content_pillars) + 合集/置顶建议(collections)。
5. 合规：不真人肖像、不引战、不盗版侵权、符合该平台社区规范。

【输出】严格输出 JSON 数组：
{
  "platform": "抖音",
  "persona": "童年游戏考古号",
  "nickname": "...",
  "nickname_candidates": ["...", "...", "..."],
  "bio": "...",
  "avatar_prompt": "...",
  "profile_plan": { "content_pillars": ["...","...","..."], "collections": ["...","..."] }
}
只输出 JSON。
```

### 6.2 内容生成（按平台形态改造）

```text
母题：【魔法哈奇/哈奇小镇 童年怀旧 · 90末-00后】
选题：<在此填一个母题，如"重返哈奇小镇核心场景">
请把同一选题改造成 5 个平台的内容，各自符合平台调性：

要求：真实怀旧口吻、不夸大、不侵权；魔法哈奇画面以授权素材为准。
```

## 8. 避坑指南（合规向）

1. **批量注册 / 买黑产号 → 连坐封号**：用自有实名 + 官方企业号/矩阵，一号一档；不碰成品号、不群控。
2. **刷量 / 互赞 / 外挂 → 限流封禁**：靠真实优质内容养权重；数据要真实，宁慢勿假。
