# 教案 J5｜AI 生图爆款内容生产（Vibe Marketing）

> 适用对象：HelloTalk 市场部各地区内容运营、品牌、活动运营、自营号团队
> 课程定位：把「已被市场验证的爆款图文模板」用 AI 生图 \+ 剪映流水线化生产，单篇约 8 分钟，走向全自动 Vibe Marketing
> 教案来源：本教案的真实数据、工作流、避坑点全部来自市场部内部实战记录（田丰豪 / 吴雨伦 / 沛林 在日本市场的内容测试纪要），非纸面推演
> 
> 

---

## 1\. 课程定位

### 1\.1 一句话定位

**用 AI 生图把「已经被真人 KOC 跑出过爆款」的图文模板，标准化、低成本、可批量地复刻出来，把单篇制作时间压到 8 分钟，并向「全自动 Vibe Marketing（氛围营销）」演进。**

### 1\.2 为什么是「图文向」而不是「视频向」

- 日本市场现有跨国情侣类爆款，真人创作者**多以视频形式**为主；但视频对真人出镜、配合度、单价要求高。

- 日本用户「露脸、露声音都羞耻」，真人内容的供给天然受限（雨伦反馈：日本本地人基本不愿意做假聊天记录类内容，过去只有留学生愿意做、且效果不好）。

- **AI 生图现在已经逼真到基本看不出真假**，图文方向实现成本更低、可控性更高，且对真实性要求没那么高、对「情绪与代入感」要求高 —— 这正是 AI 当前最擅长的部分。

- 结论：把验证过的视频模板**转成图文方向测试**，对比 AI 图文 vs 日本 KOC 真人内容的数据差异。

### 1\.3 什么是 Vibe Marketing（本课语境）

不靠真人出镜、不靠真实场景拍摄，**用 AI 批量生成「带情绪、带故事、看起来真实」的图文内容**，靠氛围（向往感 \+ 安全感）打动用户，配合热门音乐和字幕，规模化分发。本课目标是把这条链路从「手工打磨」推向「填空式半自动 → 全自动」。

### 1\.4 学习目标（学完能独立交付）

1. 能复用验证过的模板独立选题；

2. 能撰写 / 复用一套可复制的分镜提示词，一次性生成连贯多图；

3. 能用剪映完成配音、字幕、卡点，8 分钟产出一篇；

4. 能识别并规避 4 大坑（产品界面/Logo、人物一致性、声音情感、平台原创度）。

## 2\. 案例背景（真实播放数据 / 品类）

本课所有方法论，建立在两类**已经过市场验证、数据表现良好**的图文向内容之上。

|品类|数据表现（真实）|创作者产能|形态现状|AI 复刻可行性|
|---|---|---|---|---|
|① 跨国情侣|均播 \~5,000/篇；偶发数百万播放量爆款|用户月均产量 \~25 篇/月|日本创作者当前多为视频|高（建议转图文测试）|
|② 跨国礼物互赠|高 380k，低也有 2,000\+|过去产量不大（早期 AI 生图弱、成本高）|美国 KOC 做过尝试|高（现在 AI 已看不出真假）|

**关键判断（来自实战纪要）：**

- 跨国情侣：故事本身可延展出非常多方向，是「一个模板、无限选题」的富矿。

- 跨国礼物互赠：过去因 AI 生图能力弱、制作成本高所以产量小；**现在 AI 已强大到基本看不出真假，正是规模化复刻的窗口期。**

- 日本市场内容审美：在「向往感」（我想变强 / 我想交友）与「安全感」（我不会被骚扰 / 我不用露脸）之间取得平衡的素材才容易成功。

## 4\. 详细工作流细化

> 总链路：**选题 → 撰写/复用提示词 → AI 一次性生成带故事多图 → 剪映配音/字幕 → 人工质检 → 发布**
> 目标节拍：提示词定稿后，单篇 ≈ **8 分钟**。前期提示词打磨慢（细节多），但**一次打磨、长期复用**。
> 
> 

### 步骤 1：选题（复用验证模板）

|项|内容|
|---|---|
|操作|从模板库选定品类（跨国情侣 / 礼物互赠）；定本篇人物（国籍 CP，如 日×韩 / 日×法）、关键「物」、情绪弧（相遇→送礼→感动）。|
|角色|内容策划|
|AI 工具|用 Gemini / ChatGPT 做选题头脑风暴（生成 N 个故事梗概供挑选）|
|提示词示例|「你是 HelloTalk 日本市场内容策划。基于『跨国情侣礼物互赠』这一已验证爆款模板，产出 5 个适合日本用户的故事梗概，每个包含：男女主国籍、相识场景（在 HelloTalk 上学语言）、礼物、情绪高潮点。要求平衡『向往感』与『安全感』，避免任何骚扰/露骨元素。」|
|产出|1 个选定故事梗概（含人物卡、礼物、4–6 个分镜情绪点）|

### 步骤 2：撰写 / 复用提示词

|项|内容|
|---|---|
|操作|套用第 6 章「分镜提示词模板」，填入本篇人物/物/文字；重点打磨细节（日韩手写字体、礼物细节、文字替代界面）。**首篇慢、复用快。**|
|角色|提示词工程师（\+ 本地化顾问校字）|
|AI 工具|即梦 / 可灵 / Midjourney / Nano Banana|
|提示词示例|见第 6 章完整分镜模板（含 character sheet 锁人物）|
|产出|一套 4–6 张连续分镜的 prompt（已锁定人物一致性 token、字体、物料细节）|
|提速要点|把人物设定写成固定「角色卡」段落，每张图复用同一段，仅改动作/场景；这是后续一致性的根基。|

### 步骤 3：AI 一次性生成带故事多图（故事板）

|项|内容|
|---|---|
|操作|一次性生成带故事性的多图（而非逐张孤立生成），保证人物、画风、光线连贯；不满意的单张做局部重绘/参考图重生成。|
|角色|AI 生图操作|
|AI 工具|即梦 / 可灵（支持参考图保一致性）/ Nano Banana（多图编辑、保主体）/ Midjourney（`--cref` 角色参考）|
|提示词示例|「以下 6 个分镜为同一对情侣的连续故事，保持人物外貌、画风、色调一致：分镜1…分镜2…（接第 6 章）」|
|产出|4–6 张连贯分镜图，主角同一张脸、同一画风|

### 步骤 4：剪映配音 / 字幕 / 卡点

|项|内容|
|---|---|
|操作|多图导入剪映 → 套热门音乐 → 图片切换卡到鼓点 → 加配音 → 加字幕（日/韩原文 \+ 对照）。海外内容更偏音乐驱动。|
|角色|剪辑/配音|
|AI 工具|剪映（配音/字幕/卡点）；配音模型见第 8 章；热门音乐：TikTok Creative Center（按国家筛）、YouTube 榜单（yt\-dlp 下干净音频）|
|提示词示例|配音脚本示例（日语）：「ねえ聞いて。最近ハロートークで韓国語を勉強してるんだけど…」（注意：交给配音前先确认模型不会把汉字按中文读，见避坑）|
|产出|一条带 BGM、字幕、配音、卡点的成片|

### 步骤 5：人工质检（细节 / 品牌）

|项|内容|
|---|---|
|操作|逐张核：① 是否出现 HT 聊天界面/动态页/Logo 的生成错误；② 人物前后是否一致；③ 手写文字有无错别字/乱码；④ 礼物细节是否合理；⑤ 平台原创度（去重/像素级差异化）。|
|角色|质检 QC（\+ 本地化顾问校语言）|
|AI 工具|人工为主；可用 FFmpeg 做像素级「洗片/去重」差异化处理（提升平台原创度判定，非用于剪辑主流程）|
|提示词示例|QC checklist（见第 8 章避坑表，逐项打勾）|
|产出|通过质检的成片 / 退回重做清单|

### 步骤 6：发布

|项|内容|
|---|---|
|操作|按目标平台（日本 TikTok / IG / YouTube 等）发布；记录数据，回流对比「AI 图文 vs 真人 KOC」。|
|角色|内容运营|
|AI 工具|—|
|产出|发布链接 \+ 数据回收表（播放/互动/涨粉）|

## 5\. AI 工具：安装 / 配置 / 使用

> 本章覆盖：主流生图工具账号与使用、「一次性生成连贯多图（故事板）」技巧、剪映配音/卡点、人物一致性技巧。
> 
> 

### 5\.1 主流 AI 生图工具对照

|工具|入口 / 平台|账号注册|适用场景|一致性能力|
|---|---|---|---|---|
|即梦（Dreamina）|网页 / App（字节）|手机号/抖音号登录|中文友好、出图快、API 可接（内部数字人后台已接即梦 API）|参考图 \+ 角色锁定|
|可灵（Kling，快手）|网页 / App|手机号/快手号|图生图、图生视频强（实测抽象动物用可灵做）|参考图保主体好，API 可接|
|Midjourney|Discord / 官网|订阅制（绑卡）|画质/氛围最强|`--cref`（角色参考）、`--sref`（风格参考）|
|Nano Banana|Google 系图像编辑模型|Google 账号|多图编辑、保主体一致、局部改|强：同一主体跨图编辑|

**安装/配置通用步骤：**

1. 注册账号（即梦/可灵用手机号；Midjourney 走 Discord 订阅；Nano Banana 用 Google 账号——注意 Gemini 系账号**多人共用易被封**，优先用企业邮箱 `xxx@hellotalk.com` 登录，规避封控）。

2. 国内访问 Midjourney/Google 系需具备相应网络条件；账号集中管理、避免一号多人乱用。

3. 若走自动化：即梦/可灵均有 API，可接入内部内容中台（数字人后台已跑通「选形象→链接→文本提取→AI 纠错→生成」SOP，生图同理可接）。

### 5\.2 「一次性生成连贯多图（故事板）」技巧

目标：4–6 张图像一个连续故事，而不是 6 张互不相干的图。

1. **先定角色卡，再定分镜**：把主角外貌写成一段固定描述（character sheet），每个分镜 prompt 都带这段。

2. **首图定基准**：先生成满意的第 1 张作为「锚图」，后续用它当参考图（reference image）/ `--cref` 生成其余分镜。

3. **统一风格关键词**：所有分镜共享同一组风格词（光线、色调、镜头、画风），只改动作和场景。

4. **批量出图 \+ 挑选**：每个分镜出 4 张候选，挑最一致的；不一致的用「参考图重生成」或局部重绘修。

5. **保持构图节奏**：分镜按情绪弧排列（相遇→铺垫→礼物→高潮→收尾），便于剪映卡点。

### 5\.3 剪映：配音 / 卡点

- **导入**：多图按分镜顺序导入时间轴。

- **套热门音乐**：先按国家拿热门 BGM（TikTok Creative Center 可按日本筛选试听；YouTube 榜单用 yt\-dlp 下干净音频，跳广告），把图片切换点卡到鼓点/重拍。

- **配音**：用挑选过的声音模型（见第 8 章）生成旁白，导入对齐；情感不足的模型不要用。

- **字幕**：日/韩原文 \+ 对照字幕；手写感正文用图内手写字体，剪映字幕做翻译对照。

- **卡点逻辑**：图片停留时长 = 与音乐节拍对齐（可先用脚本生成卡点时间，再进剪映精修）。

### 5\.4 人物一致性（同一人物多图）技巧 —— 重中之重

|方法|工具|怎么做|
|---|---|---|
|角色卡固定文本|全部|每张图复用同一段外貌描述（发色/脸型/穿搭/年龄）|
|参考图（锚图）|即梦/可灵/Nano Banana|用第 1 张满意图当参考，生成后续|
|`--cref` 角色参考|Midjourney|在 prompt 后接锚图 URL \+ `--cref`|
|同主体多图编辑|Nano Banana|直接在同一主体上改场景/动作，保脸不变|
|种子/seed 固定|支持的工具|固定 seed 降低随机漂移|
|局部重绘|即梦/可灵|只重绘脸或手等出错区域，不动整图|

## 6\. 可复制的「跨国情侣故事图文」分镜提示词模板

> 含：人物一致性（角色卡）、日韩手写字体、礼物细节、**文字替代界面**（不画 HT 真实 UI/Logo，改用「文字性表达」）。
> 用法：先填【角色卡】，每个分镜复用角色卡，只改【场景/动作/文字】。出图后用锚图保一致性。
> 
> 

### 6\.1 角色卡（每张图都带，锁定一致性）

```
[CHARACTER SHEET — 每张分镜都粘贴这段]
女主：日本女生，22岁，齐肩黑直发，刘海偏分，鹅蛋脸，淡妆，米色针织衫，气质温柔内敛。
男主：韩国男生，24岁，黑色短发，干净清爽，浅灰卫衣，笑容温和。
画风：日系清新写真感，柔和自然光，胶片颗粒，暖色调，浅景深，竖构图 9:16。
一致性：两人外貌、发型、穿搭、画风、色调在所有分镜中保持完全一致。
负向：不出现任何 App 界面、聊天软件 UI、品牌 Logo、水印、多余文字乱码、畸形手指。
```

### 6\.2 分镜 1：相遇（在 HelloTalk 学语言，但用「文字」表达，不画界面）

```
[粘贴角色卡]
分镜1：女主坐在窗边咖啡桌前看手机，温柔微笑。手机屏幕只显示一行手写感文字便签风格：
「韓国語、教えてくれてありがとう」（谢谢你教我韩语），手写日文字体，自然贴在画面里。
不要画任何聊天软件界面、对话气泡 UI 或 Logo；用桌上一张手写便签纸表达「在语言交友中相识」。
暖光，窗外虚化街景，胶片质感，9:16。
```

### 6\.3 分镜 2：铺垫 / 期待

```
[粘贴角色卡]
分镜2：男主在韩国的房间里，桌上摊开一张手写信纸，正在认真书写。
信纸上可见手写韩文字体若干行（如：첫 선물, 잘 어울렸으면 좋겠다 —— 第一份礼物，希望你喜欢）。
桌面有牛皮纸、麻绳、待包装的小礼盒。暖色台灯，温馨，9:16。强调手写字体的真实感与笔触。
```

### 6\.4 分镜 3：礼物（细节是灵魂）

```
[粘贴角色卡]
分镜3：特写一份正在打包的跨国礼物：日式风吕敷布包裹的木盒，内含和菓子、一枚护身符（御守）、
一张手写卡片，卡片上手写日文「いつもありがとう」。牛皮纸＋干花＋麻绳，礼物细节丰富真实，
材质质感清晰（布纹、纸纹、绳结）。柔光俯拍，温暖治愈，9:16。不出现任何快递面单 Logo/品牌。
```

### 6\.5 分镜 4：收到礼物 / 情绪高潮

```
[粘贴角色卡]
分镜4：女主在家中拆开礼物，双手捧着护身符，眼眶微红、感动微笑。礼物盒与手写卡片在画面中。
情绪真实克制（向往感＋安全感，不夸张）。暖光，浅景深聚焦人物表情，胶片质感，9:16。
```

### 6\.6 分镜 5：回应 / 跨国呼应

```
[粘贴角色卡]
分镜5：女主也在写回礼卡片，手写日文＋简单韩文「ありがとう / 고마워」。
桌上摆好准备寄回的日本手信（抹茶点心、明信片）。画面呼应分镜2/3，构图对称，
表达「跨国互赠」的双向情感。暖色，温柔，9:16。手写字体清晰真实。
```

### 6\.7 分镜 6：收尾 / 升华（文字替代界面收口）

```
[粘贴角色卡]
分镜6：画面下方一张手写便签作为收尾文案，手写日文：
「言葉が私たちを繋いだ」（是语言把我们连在了一起）。
背景为两人各自城市的暖色窗景拼接/虚化，情绪收束，9:16。
全程不出现任何 App 截图、聊天界面或品牌 Logo —— 用手写文字承载「跨国语言社交」的主题。
```

### 6\.8 模板复用说明

- **铁律：凡涉及 HelloTalk 的地方，一律用手写文字 / 便签 / 信件表达，绝不让 AI 画聊天界面或 Logo**（详见第 8 章避坑）。

## 8\. 避坑指南

### 8\.1 产品界面 / Logo 生成出错 → 用「文字替代」

- **现象**：AI 生图涉及 HelloTalk 具体场景（聊天界面、动态页、Logo）时，几乎必出细节错误（错位、错字、Logo 变形）。

- **对策**：**用 HelloTalk 的「文字性表达」替代具体产品界面** —— 改用手写信、便签、手机里的纯文字，把「在 HT 学语言/交友」用文字叙述出来。生成稳定性大幅提升，也更适合走全自动 Vibe Marketing。

- **铁律**：prompt 负向词里写死「不出现任何 App 界面 / 聊天 UI / 品牌 Logo / 水印」。

### 8\.2 人物一致性

- **现象**：多图里主角换脸、换发型、画风漂移。

- **对策**：角色卡固定文本 \+ 锚图参考（reference / `--cref`）\+ 固定 seed \+ 局部重绘；优先用 Nano Banana / 可灵的「保主体」能力。详见 5\.4。

### 8\.3 声音情感不足 / 读音错误 → 选模型

- **现象**：

    - minimax：声音「过于完美」，情感起伏不够，偏机械正经。

    - Seedance：日语发音会把「最近」「理想」等汉字词**按中文发音读**，而非日语读法；表情也偏机械、语调 AI 味重。

- **对策**：

    - 不盲选「最清晰」的模型，要选**有情绪起伏、偏自然/真实感**的（日本用户更吃这一套）。

    - 日/韩配音必须由**懂该语言的人试听校验读音**（不懂语言时不要自行判断）。

    - 测试法：把需求丢给 Gemini/ChatGPT 让其推荐候选模型，逐个试听、对比，挑情感最自然的。

    - 双 Agent 思路：内容 Agent（控内容）\+ 语气 Agent（控语气/身份），分别调提示词。

### 8\.4 平台原创度

- **现象**：AI 图文同质化、平台判重，限流。

- **对策**：模板复用但**人物/物/文字差异化**；必要时用 FFmpeg 做像素级「洗片/去重」（把素材做差异化处理，提升原创度判定，非用于剪辑主流程）；矩阵分发时打散发布节奏。

### 8\.5 账号封控（配套坑）

- **现象**：Gemini / Google 系账号**多人共用易被封**。

- **对策**：用企业邮箱（`xxx@hellotalk.com`）登录更稳定；账号集中管理，历史数据迁到企业邮箱，避免反复封控。

### 8\.6 QC Checklist（质检逐项打勾）

```
[ ] 全篇无 HT 聊天界面/动态页/Logo 的生成错误（已用文字替代）
[ ] 主角脸/发型/穿搭/画风在所有分镜一致
[ ] 手写日/韩文字无错别字、无乱码（已由本地化顾问校验）
[ ] 礼物/物料细节真实合理（材质、绳结、卡片）
[ ] 配音有情感、读音正确（日语汉字未按中文读）
[ ] 字幕对照正确、卡点对齐音乐
[ ] 平台原创度处理完成（差异化/去重）
```
