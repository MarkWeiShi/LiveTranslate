## 0. 一句话产品定义

> **一个所有人用各自母语就能同局博弈的实时语音狼人杀**：AI 当"上帝"（控流程 + 多语言旁白 + 黑话注解 + 冷启兜底当 AI 玩家），实时翻译让巴西、印度、CIS、MENA 的玩家在同一局里推理、悍跳、对峙——而这是 Discord（锁英语）、Yalla（锁单语区）、TG 狼人 bot（纯文字单语言）结构上都做不到的。

---

## 1. 为什么是狼人杀（机会与可行性）

**机会**：狼人杀是纯语言博弈游戏，今天 100% 被语言锁死 → 翻译解锁后是"全球同局"的独有形态（母文档 §3 旗舰判断）。

**可行性（关键洞察）**：**狼人杀的核心机制天然适配实时翻译**——
- ✅ **回合制、轮流发言**：发言阶段永远只有 1 人开麦，规避自由语聊的语音重叠（重叠是实时翻译最大杀手）。
- ✅ **结构化阶段**：夜晚（私密行动）→ 白天（顺序发言 → 投票），每阶段语境明确，翻译可注入"当前是发言/投票"上下文提升准确率。
- ✅ **发言时长有上限**：每人 45–90s，天然给翻译的延迟与成本封顶。
- ✅ **术语高度有限**：角色词 + 黑话是封闭集合，可用术语表锁死一致性。

**结论**：狼人杀不是"硬塞翻译进去"，而是它的玩法结构本身就是翻译落地的最佳容器。建议作为母文档 P1 的差异化王牌重点投入。

## 2. 目标与非目标

**目标**
- G1 让**跨母语**玩家完成一局完整、公平、可推理的狼人杀（北极星：完成的跨语言对局数）。
- G2 翻译延迟与质量达到"不破坏推理节奏"（发言字幕首字 <1s，整段 <2s）。
- G3 AI 主持把"开局即冷场/凑不齐人/不懂规则"三大死穴一次性解决（含 AI 玩家兜底）。
- G4 复用 LinkU WebRTC + 翻译原型，工程增量可控；接 TG Mini App（initData 登录 + Stars 变现）。

**非目标（本期不做）**
- ✗ 自由语音抢话对喷（违翻译可行性；用结构化"抢麦队列"替代）。
- ✗ 复杂高阶板子（如 12 人禁言狼 / 骑士 / 通灵师全家桶）——首期只做标准与新手两块板。
- ✗ 竞技段位天梯（后置 P2）。
- ✗ naughty/擦边内容（母文档红线）。

## 3. 玩法规则设计

### 3.1 板子（首期两块）

| 板子 | 人数 | 配置 | 适用 | 优先级 |
|---|---|---|---|---|
| **新手屠边板** | 6 人 | 2 狼 + 1 预言家 + 1 女巫 + 2 平民 | 跨语言/新人快速局，~8–12 分钟 | P0 |
| **标准 9 人板** | 9 人 | 3 狼 + 预言家/女巫/猎人 + 3 平民 | 标准局，~15–25 分钟 | P0 |
| （后置）警长标准板 | 9/12 | 上述 + 警徽流（1.5 票/优先发言） | 进阶 | P1 |

> 凑不齐真人时由 **AI 玩家**补位（见 §5.4）；新手板更短、容错高，作为跨语言首局默认。

### 3.2 角色能力（标准定义，术语表锁定多语言译名）

| 角色 | 阵营 | 能力 |
|---|---|---|
| 狼人 Werewolf | 狼 | 夜晚集体选 1 人击杀；可"自爆"（白天强制结束发言进黑夜，P1） |
| 预言家 Seer | 神（好人） | 每夜查验 1 人身份（狼/好人） |
| 女巫 Witch | 神（好人） | 一瓶解药（救当晚被杀者）+ 一瓶毒药，全局各 1 次；同夜不可同时用（可配置） |
| 猎人 Hunter | 神（好人） | 死亡（被杀/被票出，被毒除外）时可开枪带走 1 人 |
| 平民 Villager | 好人 | 无技能，靠发言投票 |

### 3.3 流程状态机（单局）

```
准备 LOBBY → 发牌 DEAL → [夜晚 NIGHT → 天亮公布 DAWN → 发言 SPEAK(逐位) → 投票 VOTE → 放逐 EXILE → 胜负判定 CHECK] 循环 → 结算 RESULT
```

- **夜晚 NIGHT**：AI 主持依次私密唤醒 狼→预→女巫（猎人无夜间行动）；各角色在**私密通道**提交行动（见 §6.2 通道隔离）。
- **天亮 DAWN**：AI 主持多语言公布死亡（不公布身份与死因细节，除非规则要求）。死者可留"遗言"1 轮（P1）。
- **发言 SPEAK**：从随机/指定位起，逐位开麦发言（核心翻译阶段，§6.1）。
- **投票 VOTE**：同步投票，可弃票；平票进入 PK 申辩再投或同归于尽（可配置）。
- **胜负 CHECK**：狼全出局→好人胜；狼数≥好人数（或屠边：神全灭/民全灭）→狼胜。

### 3.4 麦位与参与
- 9 麦位（玩家）+ N 观战席；观战可文字 + 打赏，不可获私密信息。
- 发言阶段**强制单麦**：仅当前发言者麦开，其余静音（这是翻译可行性的硬保证）。
- **抢麦/申辩队列**（替代自由对喷）：对峙时玩家举手，AI 主持按队列发放 15s 反驳窗口，仍保持单人发言。

## 4. AI 主持（"上帝"）设计

AI 主持是产品灵魂，承担 4 个角色叠加：**流程裁判 + 多语言旁白 + 黑话教练 + 冷启兜底**。

### 4.1 职责
1. **流程控制**：推进状态机、计时、收集夜间行动、判定死亡/胜负、处理平票 PK。
2. **多语言旁白**：每个阶段播报同时生成**每位玩家母语**版本（字幕 + TTS 音色），如"天黑请闭眼"按各自语言呈现，营造氛围。
3. **节奏管理**：发言倒计时、温柔催促沉默玩家、超时自动"过"、防止单人拖场。
4. **新手黑话教练**：新人首次遇到"悍跳/查杀/银水/归票"等黑话时，在其母语侧弹出**轻量注解卡**（不打断游戏）。
5. **冷启兜底**：人数不足时以 AI 玩家补位（§5.4），并在房间级披露"本局含 AI 玩家"。

### 4.2 主持脚本与人设
- 多套主持人设/音色（沉稳法官、悬疑叙事、欢乐综艺），音色为变现点（§8）。
- 旁白脚本模板化 + LLM 润色：固定流程播报走**模板**（确定性、低延迟、零幻觉），氛围/调侃走 LLM。
- **零作弊保证**：AI 主持掌握全局真实身份，但其对外输出经"信息过滤层"——只能播报规则允许公开的信息，私密身份永不进入公共翻译/播报通道（§6.2）。

### 4.3 容错
- 玩家掉线：托管 AI 接管该位（保持牌局完整），回归后归还。
- 行动超时：狼空刀/神不发动技能按规则默认处理。
- 翻译失败降级：TTS 失败→纯字幕；MT 失败→展示原文 + 重试 + 主持口头提示。

## 5. 实时翻译交互设计（技术与体验核心）

### 5.1 发言阶段翻译流（皇冠功能）

单人发言 → 流水线 → 每位听者各自母语呈现：

```
发言者母语语音
  → 流式 STT（边说边转写）
  → MT（注入：狼人术语表 + 当前阶段上下文 + 忠实翻译指令）
  → 每位听者：① 实时字幕（主，挂发言者头像旁，原文/译文可切）
              ② 可选 TTS 配音（次，可关，默认开/关由房型定）
```

**延迟预算**：字幕首字 < 1.0s；整句 < 1.5s；TTS 配音 < 3s（异步追赶，落后时以字幕为准）。**字幕是推理主通道，TTS 是氛围增强**——因为推理要速度，字幕快于配音。

### 5.2 三大翻译难点与对策

| 难点 | 说明 | 对策 |
|---|---|---|
| **黑话一致性** | 悍跳/银水/查杀/踩/站边/归票/包赔/划水 等无标准外语对应 | **双层术语表**：①系统词（角色/阶段）固定锁译；②黑话词映射到"解释性译法 + 原词 tooltip"，AI 主持按听者母语给注解卡（§4.1.4） |
| **欺骗线索保留** | 翻译不能"修正谎言"或抹平迟疑/矛盾，否则毁掉推理 | MT 指令=**忠实翻译，保留语气/犹豫/自信/前后矛盾，不消歧义、不补全、不注入任何身份信息**；翻译服务对公共发言**身份盲**（拿不到真实角色） |
| **语音重叠** | 自由对喷无法翻译 | 玩法层根治：**强制单麦 + 抢麦队列**（§3.4），从机制上消灭重叠 |

### 5.3 字幕/配音 UI 规范
- 字幕气泡：发言者头像旁，显示「译文（大）+ 原文（小，可折叠）」+ 语言标。
- 历史发言可回看（推理需复盘）：本轮发言转写存为可滚动记录。
- 听者可设"我的接收语言"与"原文/译文/双语"偏好。
- 观战席同享字幕（无私密信息）。

### 5.4 AI 玩家（冷启兜底，母文档"AI 兜底房间热度"落地）
- LLM 智能体：感知自身角色 + 公共发言转写 → 生成目标语言发言（可悍跳/可隐藏/可推理），难度可调。
- 多语言原生：AI 玩家可用任意语言发言并被翻译，强化"跨语言"体感。
- **合规处理（需法务确认）**：房间级披露"本局含 AI 玩家/AI 兜底局"徽标 + **赛后揭示哪些是 AI**；但**对局中不实时标明具体哪位是 AI**（否则推理游戏直接破坏）。这是"母文档强制 AI 披露红线"与"游戏机制"的折中，**列为待决策项**。

## 6. 公平性 / 反作弊 / 风控

| 项 | 风险 | 对策 | 优先级 |
|---|---|---|---|
| **通道隔离** | 私密身份/夜间行动经翻译或日志泄漏给他人 | 私密信息走独立加密通道，**按角色权限分端渲染**（客户端只收到本角色有权知道的信息）；公共翻译管线物理隔离真实身份 | P0 |
| 翻译夹带 | 玩家用暗语经翻译传私密信息 | AI 风控扫描发言（识别引导转账/加私联/暗号），可标记 | P1 |
| 外挂多开/换位 | 同人多账号、上帝视角 | 设备/账号指纹、单设备单座、行为异常检测 | P1 |
| 挂机/恶意掉线 | 破坏牌局 | AI 托管接管 + 信誉分惩罚 + 匹配降权 | P1 |
| 内容合规 | 跨境资金/未成年/擦边 | 实名+年龄门、礼物链路风控、AI 实时审核（继承母文档红线） | P0 |

## 7. 变现钩子（不破坏对局公平）

> 原则：**只卖外观/体验/身份，不卖战力**（不做复活卡等 P2W）。

- 🟡 角色皮肤 / 头像框 / 发言特效（Stars 购买）
- 🟡 AI 主持音色 / 人设包（Stars）
- 🟡 观战打赏 MVP 玩家、最佳推理（Stars 礼物，§母文档变现）
- 🟡 翻译特权：高级 TTS 配音音色、优先低延迟、双语字幕（免费层字幕、付费层 TTS）
- 🟡 段位赛/精选局入场（P2）、专属房订阅（Tribute 式）

## 11. 待决策项（评审需拍板）

1. **AI 玩家披露口径**（§5.4 E6-3）：房间级披露+赛后揭示 vs 全程标明——需法务确认是否满足"强制 AI 披露"红线且不破坏推理。**关键决策。**
2. **自由讨论形态**：MVP 是否完全禁止自由对喷、只用抢麦队列（推荐是，保翻译可行）。
3. **TTS 配音默认开关**：免费层是否一律字幕、付费房才开 TTS（影响 COGS 与体验）。
4. **女巫规则**：同夜能否解药+毒药并用（影响平衡，默认不可）。
5. **STT/MT/TTS 供应商**：复用 LinkU 现有链路还是按"游戏低延迟"重选（影响 E4 估算）。
6. **首发语言对**：先支持哪几种母语（建议覆盖母文档走廊：英/葡/西/印地/阿/俄）。
