# PRD｜赛道二：跨境热点内容兴趣语聊房 + 1v1 视频（实时翻译）

> 产品代号：**WorldRoom（暂名）— 获客与留存底盘**
> 版本：v1.0（PRD Draft）｜日期：2026-06-15
> 定位（一句话）：围绕全球热点话题，和不同语言的人同处一房、母语对话——零门槛的跨文化兴趣社交。
> 战略角色：作为整体产品矩阵的**安全获客底盘 + 留存核心**；用"共同兴趣"破冰，构筑天然反诈护城河，并向赛道一（1v1 深聊）输送高质量好感关系。
> 关联文档：`跨境社交产品调研_Gemini-LiveTranslate.md`、`PRD_赛道一_跨境1v1异性深度社交.md`

## 1. 背景与目标

### 1.1 背景
- 陌生社交最难的是"第一句话"。**热点内容是天然破冰器**：围绕世界杯、热门剧、游戏、AI、音乐等全球话题，跨语种用户进房即有共同语境。
- 基于兴趣的多人公开弱私密关系，**比纯交友型关系更难被杀猪盘利用**（骗局依赖快速一对一私密化），构成天然反诈护城河，且兴趣提供持续回访理由→留存更好。
- 语聊房 + 礼物变现（Yalla/MICO）+ 试用分钟数（Litmatch）是被反复验证的成熟模型；**实时翻译让"全球同房"第一次成立**。
- 与 HelloTalk 社区基因高度一致（兴趣、跨文化、UGC、语音），资源契合度最高。

### 1.2 产品目标（MVP 阶段）
| 目标 | 衡量 |
|---|---|
| 验证兴趣破冰与留存 | 进房后发言率、次日/7日留存、人均在房时长 |
| 验证按需翻译成本可控 | 单房翻译边际成本/活跃用户、翻译触发率 |
| 验证向赛道一输送 | 房内→1v1 视频转化率 |
| 验证 HelloTalk 内容/用户迁移 | Moments 话题导入效果、语伴进房率 |

### 1.3 非目标（本期不做）
大规模公会/PK 玩法、外部版权内容转载、纯陌生 1v1 随机匹配（弱化，交给赛道一）。

## 2. 目标人群（Personas）

### Persona A —「话题猎人·小薇」（核心内容消费方 / 流量基本盘）
- **画像**：18–30 岁，全球年轻人，刷热点、追剧追星、爱表达、社恐但想参与。
- **动机**：找人聊正在发生的热点、围观有趣的人、轻社交不尴尬。
- **痛点**：① 现有社交圈聊不到这些话题/同好太少；② 跨语言聊不了全球同好；③ 直接 1v1 太有压力。
- **价值**：**流量基本盘**，进房消费内容、点赞、送小礼物，部分转化为发言者。

### Persona B —「房主·Leo」（核心供给方 / 内容生产 + 变现）⭐ 关键
- **画像**：有表达欲/主持力的活跃用户、KOC、语言达人、跨文化博主（很多来自 HelloTalk）。
- **动机**：运营话题房、聚人气、收礼物分成、积累跨国粉丝与影响力。
- **痛点**：① 开房没人来（冷启动）；② 控场难、多语言难主持；③ 变现路径不清。
- **价值**：**内容与气氛的生产者**，决定房间质量与留存；需要趁手的房主工具与分成激励。

### Persona C —「边玩边练·田中」（HelloTalk 核心迁移人群）
- **画像**：HelloTalk 语言学习者，想在真实、轻松、有话题的环境里练口语并交朋友。
- **动机**：比 1v1 学习更轻松好玩；翻译兜底让"听不懂也敢上麦"。
- **价值**：**最低成本、最高契合的迁移用户**；兴趣房是 HelloTalk"学习"到"社交娱乐"的自然延伸。

### Persona D —「升温者」（房内 → 1v1 外溢）
- **画像**：在房内对某位发言者产生好感的用户。
- **价值**：向赛道一输送的**高质量、已破冰**关系；是两条赛道协同的连接点。

## 4. 与 HelloTalk 用户体系的结合

| 维度 | 复用方式 | 价值 |
|---|---|---|
| 账号体系 | HelloTalk 一键登录、画像/兴趣/语言继承 | 0 门槛进入、精准推房 |
| 内容引擎 | HelloTalk Moments 跨文化 UGC 作为"话题引子"与导流入口 | 解决话题与内容冷启动 |
| 用户导流 | 定向邀请 HelloTalk 活跃语伴/KOC 做种子房主与发言者 | 解决房间冷启动 |

## 5. 核心功能清单（含优先级）

### 5.1 话题与房间体系
- **F-TP-01（P0）** 话题中心：平台运营热点话题 + UGC 自建话题；话题可由 HelloTalk Moments 引子生成。
- **F-TP-02（P0）** 热点榜/兴趣榜：按地区、语言、兴趣分发房间。
- **F-RM-01（P0）** 兴趣语聊房（多人）：进房即看到话题、麦位、在场用户。
- **F-RM-02（P1）** 房间类型：自由麦/申请麦/嘉宾房；公开/半公开。

### 5.2 麦位制语聊 + 按需翻译（产品核心，控成本关键）
- **F-MIC-01（P0）** 麦位制：同一时刻 1–N 个麦位发言，规避多人抢话导致翻译退化。
- **F-XL-01（P0）** **按需实时翻译**：默认翻译"当前麦上发言"；非麦发言/历史发言由用户**点选翻译**——把翻译从固定成本变为可控边际成本。
- **F-XL-02（P0）** 双语字幕（原文+我的母语），可设置自动翻译语种。
- **F-XL-03（P0）** 降级策略：翻译延迟/失败自动降文字字幕并提示。
- **F-XL-04（P1）** 翻译特权（付费）：全房自动语音翻译、更低延迟、更多语种。

### 5.3 发现与推荐
- **F-DC-01（P0）** 首页推荐：基于语言/兴趣/热点的房间流。
- **F-DC-02（P1）** "你能聊得来的房"：基于 HelloTalk 语言画像的个性化推荐。
- **F-DC-03（P1）** 正在发生的热点 Push（克制）。

### 5.4 互动与变现
- **F-IT-01（P0）** 点赞/表情/弹幕，上麦申请，关注房主。
- **F-GF-01（P0）** 虚拟礼物打赏（送麦上嘉宾/房主），房主/嘉宾分成。
- **F-GF-02（P1）** VIP 订阅：进场特效、隐身、翻译特权、专属房。
- **F-GF-03（P1）** 房间任务/榜单激励活跃。

### 5.5 1v1 出口（向赛道一输送）
- **F-1V-01（P0）** 房内对某发言者发起 1v1 私聊邀请（语音优先）。
- **F-1V-02（P1）** 升级为 1v1 实时翻译视频（跳转/调用赛道一能力），关系已破冰。

### 5.6 房主工具（供给侧）
- **F-HT-01（P0）** 一键开房、设话题、抱麦/下麦、禁言、踢人。
- **F-HT-02（P1）** 房主数据看板（人气、礼物、留存）、定时开播提醒粉丝。
- **F-HT-03（P1）** 多语言主持辅助：房主发言自动翻译给全房，降低跨语言控场难度。

### 5.7 内容安全与审核
- **F-SF-01（P0）** 多语种实时审核（语音转写+翻译后文本）：涉政/涉黄/引流/未成年。
- **F-SF-02（P0）** 一键举报、房主/平台快速处置、跨产品黑名单。
- **F-SF-03（P1）** 反诱导：房内拉私下投资/转账/外部引流检测与拦截。

## 7. 关键流程细节

### 7.1 按需翻译成本控制（核心）
默认只翻译当前麦上发言（1–N 路）；观众非麦发言/历史消息按点选触发；全房自动翻译作为付费特权。**麦位制 + 按需翻译**把翻译从"全员全程固定成本"变为"边际可控成本"，是赛道二相对赛道一的成本优势所在。

### 7.2 翻译降级
延迟/失败自动降文字字幕并提示；多人抢话场景由麦位制从产品层规避。

### 7.3 冷启动策略
HelloTalk 种子房主 + Moments 话题引子 + 平台运营热点房铺底 → 形成"进房不空"的初始供给，再靠推荐分发滚动。

## 8. 指标与验收标准

- **北极星**：周活跃房间数 × 人均有效在房时长。
- **内容健康**：进房-发言转化率、房均在场人数、房主开播留存、UGC 话题占比。
- **翻译/成本**：翻译触发率、按需翻译边际成本/活跃用户、降级率、延迟 P50/P95。
- **变现**：礼物渗透率、付费 ARPU、VIP 转化、翻译特权付费占比。
- **协同**：房内→1v1 转化率、HelloTalk 导入用户进房率与留存。
- **安全**：违规处置时效、引流诈骗命中率、举报率。
